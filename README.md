# 🔐 Електронний підпис файлу

Простий набір Node.js-скриптів для генерації RSA-ключів, підписання файлів та верифікації цифрового підпису.

---
## 📌 За що відповідає цей код?
 
Цей проєкт реалізує **систему цифрового підпису** — криптографічний механізм, який дозволяє:
 
- **Підтвердити авторство** документа або файлу — тільки власник приватного ключа міг створити підпис.
- **Гарантувати цілісність** — якщо файл було змінено хоча б на один символ після підписання, перевірка виявить це і видасть помилку.
- **Забезпечити довіру без паролів** — отримувачу не потрібно знати жодних секретів, достатньо публічного ключа.

---

### Скрипти та їх задачі:
 
| Скрипт | Що робить |
|--------|-----------|
| `generateKeys.js` | Генерує два RSA-ключа (приватний + публічний) і зберігає їх у файли `.pem` |
| `signFile.js` | Читає `document.txt` і приватний ключ, створює цифровий підпис та зберігає його у `signature.sig` |
| `signatureCheck.js` | Перевіряє, чи підпис з `signature.sig` відповідає `document.txt` та публічному ключу |

---

## 📋 Можливості

- ✅ Генерація пари RSA-ключів (2048 біт) у форматі PEM
- ✅ Підписання файлу за допомогою приватного ключа (SHA-256)
- ✅ Верифікація цифрового підпису за допомогою публічного ключа

## ⚙️ Інструкція використання
> [!IMPORTANT]
> Потрібно запускати скрипти саме в такому порядку

### Крок 1: Генерація ключів
Спочатку потрібно запустити generateKeys.js щоб створити файли `public_key.pem` та `private_key.pem`.
```bash
node generateKeys.js
```

> [!WARNING]
> Зберігайте `private_key.pem` у безпечному місці. Ніколи не передавайте його іншим.

---

### Крок 2 — Підготовка документа
 
Відкрийте файл `document.txt` і додайте вміст, який хочете підписати. Це може бути:
 
- Текст договору або угоди
- Технічне завдання
- Будь-який текстовий вміст

  Приклад вмісту:
```
Я, Такий-то Такий-то, підтверджую все що сказав Такий-то. Дата: 01.01.1001
```

### Крок 3: Підписання файлу

Щоб підписати весь вміст `document.txt` виконайте:

```bash
node signFile.js
```

Створює файл `signature.sig` з цифровим підписом у форматі Base64.

---

### Крок 4: Верифікація підпису
Запустіть signatureCheck.js який перевірить документ та видасть результат на основі того чи змінювався він, чи ні
```bash
node signatureCheck.js
```
## Результат:

| Результат | Повідомлення |
|-----------|-------------|
| ✅ Успіх | `Підпис дійсний. Документ не змінено.` |
| ❌ Помилка | `Підпис недійсний! Документ змінено або ключ неправильний.` |

### Крок 5 — Передача документа
 
Щоб отримувач міг перевірити підпис, передайте йому три файли:
 
| Файл | Навіщо |
|------|--------|
| `document.txt` | Сам документ |
| `signature.sig` | Цифровий підпис |
| `public_key.pem` | Для перевірки підпису |

> [!WARNING]
>`private_key.pem` **Заборонено передавати в чужі руки.**
